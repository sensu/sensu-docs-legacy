# Sensu Checks API

List locally defined checks and request executions.

## API Definition

`/checks` (GET)
: desc.
  : Returns the list of checks.

: example url
  : http://hostname:4567/checks

: response type
  : Array

: response codes
  : - **Success**: 200 (OK)
    - **Error**: 500 (Internal Server Error)

: output
  : ~~~ json
    [
        {
            "name": "chef_client_process",
            "command": "check-procs.rb -p /usr/bin/chef-client -W 1 -w 2 -c 3",
            "subscribers": [
                "production"
            ],
            "interval": 60
        },
        {
            "name": "website",
            "command": "check-http.rb -h localhost -p /health -P 80 -q Passed -t 30",
            "subscribers": [
                "webserver"
            ],
            "interval": 30
        }
    ]
    ~~~

`/checks/:check` (GET)
: desc.
  : Returns a check.

: example url
  : http://hostname:4567/checks/chef_client_process

: response type
  : Hash

: response codes
  : - **Success**: 200 (OK)
    - **Missing**: 404 (Not Found)
    - **Error**: 500 (Internal Server Error)

: output
  : ~~~ json
    {
        "name": "chef_client_process",
        "command": "check-procs.rb -p /usr/bin/chef-client -W 1 -w 2 -c 3",
        "subscribers": [
            "production"
        ],
        "interval": 60
    }
    ~~~

`/request` (POST)
: desc.
  : Issues a check execution request.

: example url
  : http://hostname:4567/request

: payload
  : ~~~ json
    {
        "check": "chef_client_process",
        "subscribers": [
            "production"
        ]
    }
    ~~~

: response codes
  : - **Success**: 202 (Accepted)
    - **Malformed**: 400 (Bad Request)
    - **Error**: 500 (Internal Server Error)
